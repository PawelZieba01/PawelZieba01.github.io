HA_LAN_RGB_Dimmer_ESP8266

HA_LAN_RGB_Dimmer_ESP8266

An RGBW Smart Dimmer system using ESP8266 microcontroller, MQTT, and Home Assistant integration.

Overview

HA_LAN_RGB_Dimmer_ESP8266 is an IoT device built on the ESP8266 microcontroller running MicroPython. It enables remote control of RGB/RGBW LED strips via MQTT and Home Assistant. This project allows you to control color, brightness, and effects of LED lighting through MQTT messages or Home Assistant automation.

Tech Stack

  • Hardware: ESP8266 (WiFi microcontroller, e.g., NodeMCU, Wemos D1)
  • Language: Python (MicroPython on ESP8266)
  • Protocols: MQTT (remote communication), PWM (LED control), WiFi (network connectivity)
  • Integration: Home Assistant (MQTT discovery)
  • Libraries: umqttsimple (MQTT client), network utilities

Project Structure

.
├── boot.py              Initialization, configuration mode logic, WiFi connection
├── main.py              Main loop — listening for MQTT commands and updating LEDs
├── config.txt           Configuration file (WiFi, MQTT, GPIO pins)
└── index.html           Web interface for device configuration
 

How It Works

Startup & Configuration Mode

After startup, ESP8266 checks a GPIO pin. If the pin is shorted to GND, the device enters configuration mode:

  • Creates a WiFi network named ESP_RGB_Dimmer (no password)
  • Runs an HTTP server on port 80
  • Serves an HTML page (index.html) with a configuration form
  • Receives JSON configuration data and saves it to config.txt

Normal Operation Mode (Pin not shorted)

  • Loads configuration from config.txt
  • Connects to the configured WiFi network
  • Establishes connection with MQTT broker
  • main.py listens to MQTT topics for color and brightness commands
  • Updates PWM outputs to control RGB/RGBW LED strip colors and brightness
  • Supports dimming, color selection, and lighting effects

Getting Started

Step 1: Initial Configuration

  1. Upload all Python files to ESP8266
  2. Short configuration pin to GND
  3. Restart ESP8266

ESP8266 will boot in configuration mode:

  1. Connect to the ESP_RGB_Dimmer WiFi network
  2. Open your browser and navigate to http://192.168.1.1
  3. Fill out the configuration form:
    • GPIO Pins — R, G, B, W (white) channel pins
    • WiFi SSID — your network name
    • WiFi Password
    • MQTT Broker Address (e.g., mosquitto server, Home Assistant)
    • MQTT Topic — base topic for control (e.g., home/bedroom/rgb_light)
    • MQTT Username (if authentication required)
    • MQTT Password
  4. Click "Save"

Step 2: Run in Normal Mode

  1. Disconnect configuration pin from GND
  2. Restart ESP8266

The device will now boot in normal mode, connect to WiFi and MQTT. LED will blink to confirm connection.

Step 3: Control Your LEDs

Using mosquitto_pub:

# Set color to red (255, 0, 0) and brightness to 100%
mosquitto_pub -h <MQTT_SERVER> -t home/bedroom/rgb_light/set -m '{"r":255,"g":0,"b":0,"brightness":255}' -u <username> -P <password>

# Set color to blue
mosquitto_pub -h <MQTT_SERVER> -t home/bedroom/rgb_light/set -m '{"r":0,"g":0,"b":255,"brightness":255}' -u <username> -P <password>

# Dim to 50%
mosquitto_pub -h <MQTT_SERVER> -t home/bedroom/rgb_light/brightness -m '128' -u <username> -P <password>
 

Using Home Assistant:

service: mqtt.publish
data:
  topic: home/bedroom/rgb_light/set
  payload: '{"r":255,"g":0,"b":0,"brightness":255}'
 

Using Node-RED, Home Assistant, or any MQTT client:

  • Connect to your MQTT broker
  • Publish JSON messages or simple values to control:
    • Color values (RGB)
    • Brightness (0-255 PWM value)
    • On/Off commands

Configuration File Format

The config.txt file stores configuration in JSON format:

{
  "WiFissid": "your_wifi_name",
  "WiFipass": "your_wifi_password",
  "MQTTSrv": "mqtt.example.com",
  "MQTTBaseTopic": "home/bedroom/rgb_light",
  "MQTTUserName": "mqtt_user",
  "MQTTUserPass": "mqtt_password",
  "GPIO_R": 12,
  "GPIO_G": 13,
  "GPIO_B": 14,
  "GPIO_W": 15
}
 

MQTT Topics

Subscribe (device listens for commands):

  • {base_topic}/set — JSON with r, g, b, brightness
  • {base_topic}/color — HEX color (e.g., FF5500)
  • {base_topic}/brightness — Value 0-255
  • {base_topic}/on — Turn on (any value)
  • {base_topic}/off — Turn off

Publish (device reports state):

  • {base_topic}/state — Current color and brightness

Troubleshooting

  • Device won't connect to WiFi: Verify SSID and password are correct
  • MQTT connection fails: Check broker address, username, and password
  • LEDs not responding:
    • Verify GPIO pins are correct
    • Check MQTT topic matches configuration
    • Ensure PWM is working (test with digital multimeter)
    • Check LED strip power supply
  • Flickering or dim LEDs: Add capacitors (100µF) across power pins, use quality power supply

Hardware Wiring Example

ESP8266 Pin 12 (GPIO12) → MOSFET/PWM Controller (Red channel)
ESP8266 Pin 13 (GPIO13) → MOSFET/PWM Controller (Green channel)
ESP8266 Pin 14 (GPIO14) → MOSFET/PWM Controller (Blue channel)
ESP8266 Pin 15 (GPIO15) → MOSFET/PWM Controller (White channel)
ESP8266 GND → MOSFET GND, LED Strip GND
