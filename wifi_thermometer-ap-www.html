WiFi_Termometer-AP-WWW

A WiFi-enabled thermometer using ESP32 microcontroller with real-time temperature display and web interface.

Overview

WiFi_Termometer-AP-WWW- is an IoT device built on the ESP32 (ESP-WROOM-32) microcontroller running MicroPython. It operates as a wireless access point and web server, reading temperature from a DS18B20 sensor and displaying it on a responsive web interface. Users connect to the device's WiFi network and access the live temperature display through any web browser.

Tech Stack

  • Hardware: ESP-WROOM-32 (WiFi microcontroller), DS18B20 (1-wire temperature sensor)
  • Language: Python (MicroPython on ESP32), HTML/JavaScript
  • Protocols: WiFi (Access Point mode), HTTP (web server), 1-wire (sensor communication), DNS
  • Libraries: onewire, ds18x20 (sensor protocol), machine (GPIO control), socket (HTTP server), network (WiFi AP)

Project Structure

├── main.py              Main application: WiFi AP setup, HTTP server, sensor reading loop
├── microDNSSrv.py       Custom DNS server for domain resolution (MIT licensed)
├── index.html           Web UI: real-time temperature display, JavaScript polling
└── README.md            Documentation
 

How It Works

Initialization & Setup

  1. ESP32 boots and initializes as a WiFi Access Point
  2. Creates WiFi network: "Termometr WI-FI" (no password required)
  3. Assigns IP: 192.168.2.1 (subnet: 255.255.255.0)
  4. Starts DNS server that resolves any domain to device IP
  5. Starts HTTP server on port 80 listening for connections
  6. Initializes DS18B20 temperature sensor on GPIO 15
  7. Sets up LED on GPIO 2 for status indication

Normal Operation

  • Browser Connection: User connects to "Termometr WI-FI" network and navigates to:

    • http://192.168.2.1 (direct IP)
    • http://termometr.pl or any domain (resolved by DNS server)

  • Web Interface: index.html loads and starts JavaScript polling

  • Temperature Reading: JavaScript sends /getTemp request every 1 second

  • Server Response: main.py receives request, reads DS18B20 sensor, returns:

    • Current time (HH:MM:SS format)
    • Temperature value (e.g., "23.5°C")

  • Display Update: Web page updates temperature and time in real-time

  • Error Handling: If sensor is disconnected, error message blinks on page

LED Status Signals

  • Blinks every 0.4s: Waiting for WiFi device to connect
  • Blinks every 1s: Taking temperature measurement and sending data
  • Blinks rapidly (0.1s intervals): Successful WiFi connection confirmed (~2 seconds)
  • Solid on: Temperature sensor not detected
  • Off: Normal operation, waiting for browser requests

Getting Started

Requirements

  • ESP32 (ESP-WROOM-32) with MicroPython firmware installed
  • DS18B20 temperature sensor
  • LED (optional, for status indication)
  • Wiring:
    • DS18B20 data pin → GPIO 15 (configurable in main.py)
    • DS18B20 GND → GND
    • DS18B20 VCC → 3.3V (with pull-up resistor ~4.7kΩ)
    • LED anode → GPIO 2, LED cathode → GND (optional)

Step 1: Upload Files to ESP32

  1. Upload the following files to ESP32 filesystem:

    • main.py
    • microDNSSrv.py
    • index.html

  2. Ensure MicroPython is installed on your ESP32

Step 2: Power On & Connect

  1. Power on the ESP32
  2. LED will blink every 0.4s while waiting for connection
  3. Open WiFi network list on your phone/computer
  4. Connect to "Termometr WI-FI" network (no password)
  5. Open web browser and navigate to:
    • http://192.168.2.1 (recommended)
    • http://termometr.pl (or any domain)

Step 3: View Temperature

  • Temperature and time update every 1 second
  • Large display: "Temperatura: XX.X°C"
  • Time display: "HH:MM:SS"
  • If sensor is missing: Error message blinks with red text

Configuration

Adjustable Parameters in main.py

# WiFi AP Settings
ap_ip = "192.168.2.1"         # Device IP address
ap_mask = "255.255.255.0"     # Subnet mask
ap_gate = "192.168.2.1"       # Gateway
ap_ssid = "Termometr WI-FI"   # WiFi network name

# Sensor Pin
ds_pin = machine.Pin(15)      # DS18B20 data pin (change if needed)

# DNS Configuration
termo_url = "termometr.*"     # Domain pattern (e.g., termometr.pl, termometr.com, etc.)

# LED Pin
led = Pin(2, Pin.OUT)         # Status LED on GPIO 2
 

Modify RTC Time

Current code sets fixed date (2014-05-01). To use current time:

# In main.py, line 130 - replace with:
import ntptime
ntptime.settime()  # Syncs with NTP server
 

Troubleshooting

  • Device won't show up in WiFi list:

    • Verify MicroPython is properly flashed
    • Check power supply (at least 500mA recommended)

  • Browser can't connect to device:

    • Verify you're on the "Termometr WI-FI" network
    • Try both 192.168.2.1 and termometr.pl
    • Restart ESP32

  • Temperature not updating:

    • Verify DS18B20 sensor wiring (especially pull-up resistor)
    • Check GPIO 15 is not used elsewhere
    • Look for error message on page

  • Sensor always shows error:

    • Verify DS18B20 is connected to GPIO 15
    • Try higher pull-up resistor (4.7kΩ - 10kΩ)
    • Check sensor is genuine (counterfeits common)
    • Reduce cable length if using long wires

  • LED doesn't work:

    • Verify LED polarity (longer leg to GPIO 2)
    • Try different GPIO pin and update main.py
    • LED is optional and doesn't affect thermometer
