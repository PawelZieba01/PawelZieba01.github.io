Wake_On_Lan_ESP32

A remote computer wake-up system using ESP32 microcontroller, MQTT, and Wake-on-LAN protocol.

Overview

Wake_On_Lan_ESP32 is an IoT device built on the ESP32 (ESP-WROOM-32) microcontroller running MicroPython. It enables remote activation of a PC via MQTT and Wake-on-LAN protocol. This project solves the problem of remotely powering on a computer from outside its local network, bypassing the lack of a public IP address — the device acts as a communication bridge between an MQTT broker and the WoL protocol within the local network.

Tech Stack

  • Hardware: ESP-WROOM-32 (WiFi microcontroller)
  • Language: Python (MicroPython on ESP32)
  • Protocols: MQTT (remote communication), Wake-on-LAN (device activation), WiFi (network connectivity)
  • Libraries: umqttsimple (MQTT client), socket (network communication)

Project Structure

.
├── boot.py              Initialization, configuration mode logic, WiFi connection
├── main.py              Main loop — listening for MQTT commands and sending WoL packets
├── wakeonlan.py         Wake-on-LAN implementation (packet generation and transmission)
├── umqttsimple.py       MQTT client for broker communication
├── config.txt           Configuration file (WiFi, MQTT, target MAC address)
└── index.html           Web interface for device configuration
 

How It Works

Startup & Configuration Mode

After startup, ESP32 checks pin D4. If pin D4 is shorted to GND, the device enters configuration mode:

  • Creates a WiFi network named ESP_WakeOnLan (no password)
  • Runs an HTTP server on port 80
  • Serves an HTML page (index.html) with a configuration form
  • Receives JSON configuration data and saves it to config.txt

Normal Operation Mode (Pin D4 not shorted)

  • Loads configuration from config.txt
  • Connects to the configured WiFi network
  • Establishes connection with MQTT broker
  • main.py listens to the subscribed MQTT topic
  • When signal '1' is received, sends a Wake-on-LAN magic packet (wakeonlan.py) to the target device
  • LED blinks as confirmation of packet transmission

Getting Started

Step 1: Initial Configuration

  1. Upload all Python files to ESP32
  2. Short pin D4 to GND
  3. Restart ESP32

ESP32 will boot in configuration mode:

  1. Connect to the ESP_WakeOnLan WiFi network
  2. Open your browser and navigate to http://192.168.1.1
  3. Fill out the configuration form:
    • Target MAC Address of the computer to wake (format: AA:BB:CC:DD:EE:FF)
    • WiFi SSID — your network name
    • WiFi Password
    • MQTT Broker Address (e.g., mosquitto server, Home Assistant)
    • MQTT Topic — subscription topic (e.g., home/wol/pc)
    • MQTT Username (if authentication required)
    • MQTT Password
  4. Click "Save"

Step 2: Run in Normal Mode

  1. Disconnect pin D4 from GND
  2. Restart ESP32

The device will now boot in normal mode, connect to WiFi and MQTT. LED will blink twice to confirm connection.

Step 3: Wake Up Your Computer

Send message '1' to your MQTT topic:

Using mosquitto_pub:

mosquitto_pub -h <MQTT_SERVER> -t home/wol/pc -m '1' -u <username> -P <password>
 

Using Home Assistant:

service: mqtt.publish
data:
  topic: home/wol/pc
  payload: '1'
 

Using Node-RED, MQTT.fx, or any MQTT client:

  • Connect to your MQTT broker
  • Publish message '1' to your configured topic

ESP32 will send a magic packet to your local network, and your computer will power on (requires Wake-on-LAN enabled in BIOS).

Features

  • 🌐 Remote computer activation via MQTT from anywhere
  • ⚙️ Easy web-based configuration interface
  • 🔐 MQTT authentication support
  • 💡 Visual feedback via onboard LED
  • 🔄 Automatic reconnection on connection loss
  • 📦 Minimal dependencies (MicroPython standard library)

Requirements

  • ESP32 (ESP-WROOM-32) or compatible
  • MicroPython firmware
  • MQTT broker (mosquitto, Home Assistant, etc.)
  • Target computer with Wake-on-LAN enabled in BIOS
  • Same local network for target computer (ESP32 and computer must be in the same LAN)

Configuration File Format

The config.txt file stores configuration in JSON format:

{
  "WiFissid": "your_wifi_name",
  "WiFipass": "your_wifi_password",
  "MQTTSrv": "mqtt.example.com",
  "MQTTSubTopic": "home/wol/pc",
  "MQTTUserName": "mqtt_user",
  "MQTTUserPass": "mqtt_password",
  "RemoteMAC": "AA:BB:CC:DD:EE:FF"
}
 

Troubleshooting

  • Device won't connect to WiFi: Verify SSID and password are correct
  • MQTT connection fails: Check broker address, username, and password
  • Computer won't wake up:
    • Ensure Wake-on-LAN is enabled in BIOS
    • Verify target MAC address is correct
    • Check that ESP32 and computer are on the same network
  • LED doesn't blink: Verify LED is connected to GPIO2, or modify pin in boot.py
