Android Remote RC Control via UDP

A lightweight Android application that enables remote control of an RC vehicle using UDP communication protocol. The app provides an intuitive interface with dual analog joysticks and toggle buttons for basic vehicle movement and auxiliary controls.

Project Overview

This is a mobile control application built with B4A (Basic4Android) framework that communicates with an RC vehicle receiver (or any UDP-enabled device) via network sockets. The vehicle acts as a UDP server receiving control commands, while the app sends movement and state data in JSON format.

Key Features

  • Dual Joystick Control: Two independent analog joysticks for intuitive vehicle maneuvering

    • Left joystick: Vehicle power/thrust control
    • Right joystick: Steering/turning control

  • Auxiliary Buttons: Three toggle buttons for additional functions

    • Button 1, 2, 3: Custom vehicle functions (lights, special modes, etc.)

  • Battery Monitoring: Real-time battery level feedback from the remote vehicle

    • Receives battery status via UDP in JSON format
    • Displays current battery percentage

  • Network Configuration UI:

    • Manual IP address configuration
    • Custom remote port selection
    • Local listening port setup
    • Input validation (IPv4 format, port ranges 1-65535)

  • Landscape Mode: Full-screen optimized interface for comfortable landscape gameplay

How It Works

Communication Protocol

  1. Initialization Phase:

    • User enters remote device IP address and port in the main configuration screen
    • App validates IP format and port ranges
    • Proceeds to the control screen upon successful validation

  2. Control Transmission:

    • Joystick and button inputs are captured in real-time
    • Data is packaged into JSON format with the following structure:
      {
  "Pos_X": -100 to 100,      // Steering input
  "Pos_Y": -100 to 100,      // Power/speed input
  "Btn1_state": true/false,  // Button 1 toggle state
  "Btn2_state": true/false,  // Button 2 toggle state
  "Btn3_state": true/false   // Button 3 toggle state
}
       
    • JSON payload is sent via UDP to the configured remote address and port

  3. Status Reception:

    • App listens for UDP packets on configured local port
    • Receives battery status from remote device in JSON format:
      {
  "Battery_level": 0-100
}
       
    • Displays battery percentage in real-time on the control screen

Technical Details

  • Framework: B4A (Basic4Android) - Cross-platform rapid development framework
  • Protocol: UDP (connectionless, low-latency)
  • Data Format: JSON (human-readable, easily extensible)
  • Orientation: Landscape only (optimized for vehicle control)

Usage

  1. Configure Connection:

    • Enter the IP address of your RC vehicle/receiver
    • Specify the remote port the vehicle is listening on (default: 1900)
    • Set a local port to receive battery feedback (can be 0 for auto-assignment)
    • Tap "Connect" to proceed

  2. Control Vehicle:

    • Use left joystick to control forward/backward movement
    • Use right joystick to control steering left/right
    • Toggle Btn1, Btn2, Btn3 for auxiliary functions
    • Watch battery level in real-time

  3. Return to Configuration:

    • Tap the back button to return to the connection screen
    • Modify settings and reconnect as needed

Input Validation

The app includes robust input validation:

  • IP Address: Standard IPv4 format validation using regex
  • Remote Port: Range 1-65535
  • Listen Port: Range 0-65535 (0 = auto-assign)

Dependencies

  • B4A core library
  • Network library (UDP support)
  • JSON library (data serialization)
  • B4XPages library (UI page management)
  • Joystick wrapper library (analog control)
  • Collections library (data structures)

Potential Applications

  • RC car remote control
  • Drone/quadcopter control
  • Remote robot operation
  • IoT device manipulation
  • Network-based device control experiments

Notes

  • Uses UDP for low-latency wireless communication
  • JSON protocol makes it easy to extend with additional sensors or controls
  • Can be adapted for any UDP-enabled remote device
